First create() should fully load the model

When calling LanguageModel.create(), the setup and loading of the model should be fully completed. However, a bug causes the first call to promptStreaming() on the initial session to bear the cost of some delayed initialization, making its Time to First Token (TTFT) significantly slower than subsequent sessions.

If the bug is present, Session 1's TTFT will take significantly longer than Session 2's TTFT. The Time to First Token (TTFT) should be similar from the first call to the next.

Repro Code 
const session1 = await LanguageModel.create();
const stream1 = session1.promptStreaming("Who are you?");
for await (const chunk of stream1) {{ '{' }} // ⚠️ Slow! Delayed init happens before first token.
  break;
{{ '}' }}

const session2 = await LanguageModel.create();
const stream2 = session2.promptStreaming("Who are you?");
for await (const chunk of stream2) {{ '{' }} // ⚡ Fast TTFT.
  break;
{{ '}' }}
